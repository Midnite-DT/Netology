# Netology
Задача 2:

высоконагруженная база данных MySql, критичная к отказу – физические сервера. Отсутствие лишних слоёв абстракций для обеспечения любого вида программной виртуализации позволит использовать большее количество ресурсов для решения задач с БД, плюс обращение ВМ напрямую к аппаратной части сервера исключает все проблемы, которые существуют у программных решений, связанные с зависимостью от хостовой ОС. 

различные web-приложения – виртуализация уровня ОС. Контейнеры позволят, во-первых, более оперативно работать с веб-приложениями + она легко масштабируется, что может быть полезно в случае с такой нестабильной субстанцией, как веб-приложения. Во-вторых, при виртуализации уровня ОС, гостевые ОС можно модифицировать, отрезав им лишний функционал, не нужный для веб-приложений, что сэкономит ресурсы. 

Windows-системы для использования бухгалтерским отделом – паравиртаулизация. Hyper-V нативнее всего подойдёт для этой задачи, а программный гипервизор проще и дешевле развернуть, для бухгалтерии аппаратная виртуализация не целесообразна
системы, выполняющие высокопроизводительные расчёты на GPU – физические сервера. Чтобы упомянутые системы высокоэффективно выполняли свои высокопроизводительные расчёты, необходимо чтобы они могли образаться к gpu напрямую, поэтому здесь необходимо аппаратное решение.

Задача 3:

1.	VMware vSphere (ESXi). Для ста машин с разными осями лучше подойдёт аппаратная виртуализация, а упомянутый продукт умеет в балансировку и репликацию и прекрасно интегрирует Veeam Backup & Replication для резервного копирования.

Задача 4:

Гетерогенная среда:
1.	Дорого
2.	Сложность в управлении
3.	Сложность в мониторинге
4.	Сложность при миграции
5.	Сложность при масштабировании
6.	Сложность при обновлении
7.	Сложность с совместимостью
8.	Сложность с безопасностью

Как минимизировать риски:
1.	Обучение администрирующего персонала
2.	Внедрить системы мониторинга, регулярно проводить аудит.
3.	Внедрить системы централизованного управления
4.	Унифицировать политики безопасности
5.	Тщательное планирование и тестирование
6.	Системы мультиплатформенного резервирования

По возможности связываться с гетерогенными средами не стоит из-за всего вышеупомянутого, но и применяется такое решение лишь в ситуациях исключительной необходимости в нескольких разных гипервизорах одновременно.
